# 發票機串接架構設計（WinPOS WP-560 二聯式發票機）

> 目標：把現有「每日結帳手動抄發票號碼」升級為**系統直接驅動實體發票機列印統一發票**，並在收款當下自動記下發票號、結帳起訖號自動帶出。
> 決策：**走實體紙本統一發票（二聯式），不走電子發票 / 加值中心 API**（使用者拍板 2026-07-21）。
> 狀態：**架構討論階段**，尚未動工。動工前提＝取得 WP-560 原廠串接指令手冊（見第 8 節）。

## 0. 現況（2026-07 快照）

- 系統**本身不開發票**。實體發票機印統一發票，店員在**每日結帳**（`DailySettlementPage`）手動輸入發票起訖號、作廢號、作廢金額。
- 系統僅收下號碼供報表：`dailySettlements` 存 `invoiceSegments`(多段起訖) / `invoiceStartNumber` / `invoiceLastNumber` / `invoiceVoidNumbers` / `voidInvoiceAmount`；產「月銷售紀錄」與「發票明細表」（仿二聯式收銀機統一發票明細表）。
- 收款點（未來要掛開票）目前都已呼叫 `recordTransaction()` 記營收：
  | 流程 | 收款確認點（員工端） | 交易 type |
  |---|---|---|
  | 入場 | `CheckinPage` 掃碼→確認入場／電話入場／已付費放行 | `checkin` |
  | 租借 | `RentalsPage` 確認收款/取件 | `rental` |
  | 銷售 POS | `SalesPage` 結帳 | `product` / `productSales` |
  | 課程 | 待辦頁待收款確認（現金/轉帳）／報名現金 | `course` |
  | 定期票 | `POST /passes`／入場 buy_pass | `pass` |

## 1. 硬體：WinPOS WP-560（已查證可串）

- **機型**：WinPOS WP-560 **二聯式發票印表機**（配件性質，需搭主機/PC 驅動；非獨立收銀機）。
- **列印方式**：**9 針點陣撞擊式**（impact，非熱感）→ 適合二聯式複寫紙；速度約 3 行/秒。
- **介面**：**RS-232 序列埠** ＋ 並列埠。現代 PC 需 **USB-to-serial 轉接線**。
- **指令集**：**ESC/POS、TP-3688、HPC-2000、IPT-2009 多模式，用 DIP 開關切換** → 走 **ESC/POS（公開標準指令集）** 即可程式驅動（關鍵，不是只能手按的機型）。
- **能力**：二聯式（存根聯＋收執聯）同時列印、證明聯單獨列印、自動定位/裁切、缺紙/卡紙/定位錯誤警示、可接錢櫃、內建中文字庫（16×9 點陣）。
- 參考：momo `i_code=8386654`、大山事務機 `dasan.com.tw/product/wp-560/`、百揚 Ushow POS、RS232 線材 winsmaker（WP520/WP560/WP103S 專用）。

## 2. 關鍵現實：紙本發票的「號碼」怎麼管

- 二聯式紙本統一發票的**號碼是預先印在財政部配號的專用發票紙捲上**。WP-560 只做「**自動定位**」：把日期/金額/品項印到當前那張預印號碼的發票上，**機器本身不產生號碼**。
- 因此號碼由**系統計數**管理：
  1. 店員**換紙捲/換期別**時，把該捲**起始發票號**輸入系統一次（含字軌）。
  2. 每成功列印一張 → 系統把號碼記到該筆交易、**計數 +1**。
  3. **作廢**（打錯）→ 該號標作廢、跳號。
  4. 每日結帳起訖號**自動帶出**（取代現手動抄號）；月銷售/發票明細自動產生。
- ⚠️ 風險：系統計數與實體紙捲**必須對齊**。要有「目前號碼」顯示 + 換捲重設 + 對不上時可手動校正的機制。

## 3. 橋接：RS-232 → 網頁員工端（要拍板）

員工端是網頁（Firebase Hosting），無法直接驅動序列裝置。兩條路：

| 方案 | 做法 | 優點 | 缺點 |
|---|---|---|---|
| **A. 瀏覽器 Web Serial API**（建議先試） | Chrome 直接開序列埠、送 ESC/POS bytes | 不用裝程式；WP-560 就是 ESC/POS+序列，天生適合 | 限 Chrome；每台首次要授權序列埠；需維持連線 session |
| **B. 本地列印代理程式** | 櫃檯 PC 裝小程式開 COM 埠、開 `localhost` HTTP 給網頁呼叫、翻譯成印表機協定 | 最穩、任何連線方式都吃、可回報狀態 | 兩館櫃檯各要裝/維護一支程式 |

- **建議**：WP-560 先評估 **方案 A（Web Serial）**，成則最省事；不穩再退 **方案 B**。
- 兩案的「列印介面」抽象一致（見第 5 節），之後換方案只換底層傳輸。

## 4. UI 整合（二聯式較單純）

- 二聯式＝**B2C，無載具/捐贈** → 收款確認流程乾淨。
- 做一個共用流程 **「收款確認 → 送印 → 記號碼」**，四頁共用：

  `入場/租借/銷售/課程 收款確認` → 組發票內容（日期＋金額＋品項；租借**只租金不含押金**；**選填統編**） → 送 ESC/POS 列印 → WP-560 印在當前發票 → 系統記該張號碼＋計數 +1。

- **統編（選填）**：二聯式**有時客人也要印統編**（報帳） → 收款確認的共用元件加**選填統編欄**（8 碼），有填就印在發票上、存進 `invoices.buyerTaxId`。多數 B2C 留空。（真正 B2B 三聯式格式日後要再擴充。）
- **作廢連動**：現有取消/退費流程（取消入場、課程退費、試上取消…）→ 送作廢/標號碼作廢（現手動記作廢號 → 自動）。
- **補印**：交易/會員紀錄旁加「補印發票」（重印證明聯，不佔新號）。
- **開票失敗退路**：款已收、列印失敗（缺紙/斷線）→ 進「**待開立佇列**」，不卡收款；UI 顯示「發票待補印」，排除後補印。

## 5. 抽象設計（比照金流 adapter 模式）

### 5.1 前端列印介面（傳輸無關）
```
InvoicePrinter.print({ items, total, date, invoiceNumber }) → { ok, printedNumber }
InvoicePrinter.void(invoiceNumber) → { ok }
InvoicePrinter.status() → { connected, paperOk, currentNumber }
```
- adapter：`webserial`（方案 A）／`localagent`（方案 B）／`mock`(dev)。
- 共用元件 **`InvoiceCheckout`**：四頁收款確認共用，內含「送印 + 記號碼 + 失敗進佇列」。

### 5.2 後端資料模型：新增 `invoices` collection（草案）
```
invoices/{id} {
  number, track,             // 發票號碼、字軌
  gymId, date,
  amount,                    // 開立金額（不含押金）
  items: [{name, qty, price}],
  buyerTaxId,                // 選填統編（二聯有時要印；B2C 留空）
  relatedType, relatedId,    // checkin / rental / product / course / pass
  status: 'issued'|'void',
  voidReason, voidedAt,
  reprintCount,
  createdBy, createdAt,
}
```
- 每館一組「**目前發票號碼狀態**」：`gyms/{gymId}.invoiceState { track, currentNumber, rollStart, updatedAt }`（換捲重設）。
- 每日結帳改**讀 `invoices`** 算起訖/作廢，取代手動輸入（保留手動校正欄位當退路）。

### 5.3 收款點接線（同金流 orderType 對照）
| 流程 | 觸發點 | 開票金額來源 |
|---|---|---|
| 入場 | `confirmCheckIn` / `/checkin/phone` / `/direct` | `amountPaid`（含加購） |
| 租借 | `/rentals/:id/confirm`（現金）/ transfers rental 確認 | `totalRentalFee`（**排除押金**） |

## 5.4 SDK / 指令手冊現況（2026-07-21 查證）

- WP-560 走 **ESC/POS 標準指令集**（另可 DIP 切 TP-3688/HPC-2000/IPT-2009）。ESC/POS 是**公開標準**（`ESC` 0x1B / `GS` 0x1D 起始）→ **基本列印（文字、金額、對齊、切紙、開錢櫃）可直接依公開 ESC/POS 手冊實作**，不需原廠 SDK。
- 但**沒有在網路上找到 WP-560 官方完整 SDK / 程式手冊**。真正要跟原廠/經銷商拿的是：**二聯式統一發票的「定位、存根聯/收執聯、對位/裁切」細節與 DIP 設定**（每台發票紙格式的定位參數）。這部分公開 ESC/POS 手冊不涵蓋。
- 社群有用 ESC/POS 驅動同系列 WinPOS 出據機的實例（Delphi × ESC/POS × WP-T810、POS 列印電子發票證明聯）可參考做法。
- **結論**：基本列印可自幹（公開 ESC/POS）；**發票定位/複寫聯排版**建議跟經銷商要規格或現場調 DIP + 試印校正（P1 PoC 要做的事）。
