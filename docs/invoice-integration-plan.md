# 發票機串接架構設計（WinPOS WP-560 二聯式發票機）

> 目標：把現有「每日結帳手動抄發票號碼」升級為**系統直接驅動實體發票機列印統一發票**，並在收款當下自動記下發票號、結帳起訖號自動帶出。
> 決策（2026-07-21 拍板）：**實體二聯式紙本發票**（不走電子發票）＋**本地列印代理**橋接＋號碼**後端計數**＋收款當下開票＋二聯**選填統編**。
> 狀態：**架構已定，待硬體到位進 PoC**。使用者採購舊發票機＋RS-232轉USB＋二聯練習紙捲；下一步 P1＝本地代理 + 列印對位 PoC（見第 6 節）。

## 0. 現況（2026-07 快照）

- 系統**本身不開發票**。實體發票機印統一發票，店員在**每日結帳**（`DailySettlementPage`）手動輸入發票起訖號、作廢號、作廢金額。
- 系統僅收下號碼供報表：`dailySettlements` 存 `invoiceSegments`(多段起訖) / `invoiceStartNumber` / `invoiceLastNumber` / `invoiceVoidNumbers` / `voidInvoiceAmount`；產「月銷售紀錄」與「發票明細表」（仿二聯式收銀機統一發票明細表）。
- 收款點（未來要掛開票）目前都已呼叫 `recordTransaction()` 記營收：
  | 流程 | 收款確認點（員工端） | 交易 type |
  |---|---|---|
  | 入場 | `CheckinPage` 掃碼→確認入場／電話入場／已付費放行 | `checkin` |
  | 租借 | `RentalsPage` 確認收款/取件 | `rental` |
  | 銷售 POS | `SalesPage` 結帳 | `product` / `productSales` |
  | 課程 | 待辦頁待收款確認（現金/轉帳）／報名現金 | `course` |
  | 定期票 | `POST /passes`／入場 buy_pass | `pass` |

## 1. 硬體：WinPOS WP-560（已查證可串）

- **機型**：WinPOS WP-560 **二聯式發票印表機**（配件性質，需搭主機/PC 驅動；非獨立收銀機）。
- **列印方式**：**9 針點陣撞擊式**（impact，非熱感）→ 適合二聯式複寫紙；速度約 3 行/秒。
- **介面**：**RS-232 序列埠** ＋ 並列埠。現代 PC 需 **USB-to-serial 轉接線**。
- **指令集**：**ESC/POS、TP-3688、HPC-2000、IPT-2009 多模式，用 DIP 開關切換** → 走 **ESC/POS（公開標準指令集）** 即可程式驅動（關鍵，不是只能手按的機型）。
- **能力**：二聯式（存根聯＋收執聯）同時列印、證明聯單獨列印、自動定位/裁切、缺紙/卡紙/定位錯誤警示、可接錢櫃、內建中文字庫（16×9 點陣）。
- 參考：momo `i_code=8386654`、大山事務機 `dasan.com.tw/product/wp-560/`、百揚 Ushow POS、RS232 線材 winsmaker（WP520/WP560/WP103S 專用）。

## 2. 關鍵現實：紙本發票的「號碼」怎麼管

- 二聯式紙本統一發票的**號碼是預先印在財政部配號的專用發票紙捲上**。WP-560 只做「**自動定位**」：把日期/金額/品項印到當前那張預印號碼的發票上，**機器本身不產生號碼**。
- 因此號碼由**系統計數**管理：
  1. 店員**換紙捲/換期別**時，把該捲**起始發票號**輸入系統一次（含字軌）。
  2. 每成功列印一張 → 系統把號碼記到該筆交易、**計數 +1**。
  3. **作廢**（打錯）→ 該號標作廢、跳號。
  4. 每日結帳起訖號**自動帶出**（取代現手動抄號）；月銷售/發票明細自動產生。
- ⚠️ 風險：系統計數與實體紙捲**必須對齊**。要有「目前號碼」顯示 + 換捲重設 + 對不上時可手動校正的機制。

## 3. 橋接：RS-232 → 網頁員工端 → **決定走「本地列印代理」（方案 B）**

員工端是網頁（Firebase Hosting），無法直接驅動序列裝置。**已拍板走方案 B 本地代理**（2026-07-21）。

| 方案 | 做法 | 決定 |
|---|---|---|
| A. 瀏覽器 Web Serial API | Chrome 直接開序列埠送 ESC/POS | ❌ 不採用 |
| **B. 本地列印代理程式** | 櫃檯 PC 裝小程式開 COM 埠、開 `localhost` HTTP 給網頁呼叫、翻譯成 ESC/POS | ✅ **採用** |

**硬體規劃（使用者）**：買一台**舊發票機**測試、**RS-232 轉 USB** 接櫃檯電腦、已有**二聯式練習紙捲**。

### 3.1 本地代理設計
- **技術**：Node.js（與後端同語言）＋ `serialport`（開 COM 埠、送 bytes）＋ `express`（開 `http://localhost:<PORT>`）。打包成櫃檯可執行檔（`pkg`/nexe）或直接 node 跑、開機自啟。
- **端點**：
  - `POST http://localhost:<PORT>/print` — 收發票內容 → 組 ESC/POS → 寫入 COM 埠 → 回 `{ ok, error? }`
  - `GET  /status` — 連線/缺紙/COM 埠狀態
  - `POST /void`（如機型支援）／或作廢僅由後端標號碼
- **呼叫方**：員工端網頁（同一台櫃檯 Chrome）`fetch('http://localhost:<PORT>/print', …)`。
- **⚠️ 瀏覽器限制（要處理）**：HTTPS 頁（`staff.redrocktaiwan.com`）呼叫 `http://localhost`——localhost 是安全內容例外**不會被 mixed-content 擋**，但新版 Chrome 的 **Private Network Access** 會發 preflight，代理需回應 `Access-Control-Allow-Private-Network: true` ＋ CORS 允許該來源。此為已知可解的細節，代理端設定即可。
- **設定**：COM 埠編號、baud rate、DIP 對應的發票定位參數，放代理的設定檔（每館櫃檯各自設）。
- 列印介面抽象見 5.1（`InvoicePrinter` adapter `localagent`）。

## 4. UI 整合（二聯式較單純）

- 二聯式＝**B2C，無載具/捐贈** → 收款確認流程乾淨。
- 做一個共用流程 **「收款確認 → 送印 → 記號碼」**，四頁共用：

  `入場/租借/銷售/課程 收款確認` → 組發票內容（日期＋金額＋品項；租借**只租金不含押金**；**選填統編**） → 送 ESC/POS 列印 → WP-560 印在當前發票 → 系統記該張號碼＋計數 +1。

- **統編（選填）**：二聯式**有時客人也要印統編**（報帳） → 收款確認的共用元件加**選填統編欄**（8 碼），有填就印在發票上、存進 `invoices.buyerTaxId`。多數 B2C 留空。（真正 B2B 三聯式格式日後要再擴充。）
- **作廢/折讓連動**：現有取消/退費流程 → 依規則走作廢或折讓（詳見第 4.1 節）。
- **錢櫃驅動**：WP-560 有**錢櫃輸出埠**（錢櫃接發票機、非接 PC），代理用 ESC/POS `ESC p`（`0x1B 0x70`）開櫃。`InvoiceCheckout` **現金收款才帶開櫃**（電子/轉帳不開）；現金退費、每日結帳點鈔可另給「開錢櫃」鈕。代理端點加 `POST /open-drawer` 或 `/print { openDrawer:true }`。
- **補印**：交易/會員紀錄旁加「補印發票」（重印證明聯，不佔新號）。
- **開票失敗退路**：款已收、列印失敗（缺紙/斷線）→ 進「**待開立佇列**」，不卡收款；UI 顯示「發票待補印」，排除後補印。

## 4.1 退費 → 作廢 vs 折讓（合規重點）

台灣統一發票更正有**兩種合法機制、不能混用**：

- **作廢（Void）**：**三條件皆成立**才可——① 原發票**同一期別**內（統一發票以雙月為一期：7-8月、9-10月…）② **收得回買方收執聯** ③ **全額**取消。做法：兩聯註記作廢、存根留存、系統把該號碼**標作廢＋跳號**（接上結帳既有「作廢號碼/金額」）。
- **折讓（Allowance）**：**任一成立**即須折讓、不可作廢——① **跨期別**（發票已申報）② **收不回收執聯** ③ **部分退**。做法：開「銷貨退回/折讓證明單」**減當期銷售額**（不動原號碼）。

**系統判斷樹**（退費時自動判、引導店員實體動作）：
```
原發票同期別? ─否→ 折讓
全額退?       ─否→ 折讓（部分）
收得回收執聯? ─否→ 折讓
三者皆是      ─────→ 作廢（收回兩聯、跳號、仍有收款則重開）
```

**接現有退費流程**（錢的退費已由負向沖銷處理，此為加的發票層）：
| 退費流程 | 典型判定 |
|---|---|
| 取消入場（10 分/核准） | 多同日同期全額 → 作廢 |
| 課程退費 | 常跨期/部分 → 折讓 |
| 試上取消（3.88） | 同期全額→作廢；否則折讓 |
| 比賽/租借退費 | 依判斷樹 |

**落地**：作廢→`invoices.status='void'`＋帶進結帳作廢號/金額；折讓→新 `allowances` 集合（關聯原發票、折讓額）＋銷售額減項。
⚠️ **待決定**：折讓證明單怎麼開（WP-560 是二聯發票格式、折讓單格式不同）——用機器印、還是另開一份/手工單？取決於現行做法。

## 5. 抽象設計（比照金流 adapter 模式）

### 5.1 前端列印介面（傳輸無關）
```
InvoicePrinter.print({ items, total, date, buyerTaxId, openDrawer }) → { ok }
InvoicePrinter.openDrawer() → { ok }        // 現金收款/退費/點鈔（ESC p）
InvoicePrinter.status() → { connected, paperOk }
```
- adapter：**`localagent`（採用，呼叫本地代理 localhost）** ／ `mock`(dev)。
- 共用元件 **`InvoiceCheckout`**：四頁收款確認共用，內含「送印 + 記號碼 + 失敗進佇列」。
- **號碼由後端管**（機器不回報號碼；紙捲號碼預印）：後端 `invoiceState.currentNumber` 為權威；每次列印成功後端計數 +1，代理只負責把資料印到當前紙上。

### 5.2 後端資料模型：新增 `invoices` collection（草案）
```
invoices/{id} {
  number, track,             // 發票號碼、字軌
  gymId, date,
  amount,                    // 開立金額（不含押金）
  items: [{name, qty, price}],
  buyerTaxId,                // 選填統編（二聯有時要印；B2C 留空）
  relatedType, relatedId,    // checkin / rental / product / course / pass
  status: 'issued'|'void',
  voidReason, voidedAt,
  reprintCount,
  createdBy, createdAt,
}
```
- 每館一組「**目前發票號碼狀態**」：`gyms/{gymId}.invoiceState { track, currentNumber, rollStart, updatedAt }`（換捲重設）。
- 每日結帳改**讀 `invoices`** 算起訖/作廢，取代手動輸入（保留手動校正欄位當退路）。

### 5.3 收款點接線（同金流 orderType 對照）
| 流程 | 觸發點 | 開票金額來源 |
|---|---|---|
| 入場 | `confirmCheckIn` / `/checkin/phone` / `/direct` | `amountPaid`（含加購） |
| 租借 | `/rentals/:id/confirm`（現金）/ transfers rental 確認 | `totalRentalFee`（**排除押金**） |
| 銷售 | `/products/sell` | 品項合計 |
| 課程 | 待收款確認（transfers course / 現金） | 報名費 |
| 定期票 | `POST /passes` / buy_pass confirm | 票價 |

## 5.4 SDK / 指令手冊現況（2026-07-21 查證）

- WP-560 走 **ESC/POS 標準指令集**（另可 DIP 切 TP-3688/HPC-2000/IPT-2009）。ESC/POS 是**公開標準**（`ESC` 0x1B / `GS` 0x1D 起始）→ **基本列印（文字、金額、對齊、切紙、開錢櫃）可直接依公開 ESC/POS 手冊實作**，不需原廠 SDK。
- 但**沒有在網路上找到 WP-560 官方完整 SDK / 程式手冊**。社群有用 ESC/POS 驅動同系列 WinPOS 出據機的實例（Delphi × ESC/POS × WP-T810、POS 列印電子發票證明聯）可參考做法。
- **定位可自己做（靠試印校正，非必須原廠規格）**：二聯式發票的定位＝**游標控制**（換行往下 N 行、`ESC $` 絕對水平位置/空格右移 N 字）。9 針點陣固定字距，位置＝(行, 欄) 好算；發票格式財政部標準、紙捲尺寸固定 → 每欄位相對表頭的偏移量**固定、量一次寫死**；機器「自動定位」負責對齊每張表單起點，我方只管表單內欄位偏移。用**練習紙捲印→看歪→調偏移→再印**收斂即可。
- **結論**：基本列印＋定位**皆可自幹**（公開 ESC/POS + 試印校正）。經銷商的**定位範例/DIP 設定只是加速**（省初始偏移與 DIP 模式試錯），不是動工必要條件。
- ⚠️ 真正的風險點：舊機的**自動定位感測器**若異常/紙捲無定位標記，表頭對齊會較麻煩（買到機器一試即知）——這是 P1 PoC 要先確認的。

## 6. 分階段實作計畫（草案）

1. **P0 取得資源（進行中）**：使用者買**舊發票機**＋**RS-232轉USB**＋已有**二聯練習紙捲**；另向經銷商問**發票定位/DIP 設定**（ESC/POS 基本指令用公開手冊即可）。
2. **P1 本地代理 + 列印 PoC**：先做**本地代理**（Node＋serialport＋express localhost）→ 用練習紙捲印出一張二聯式測試發票，把**定位/裁切/中文**調到對位（DIP＋試印校正）。這步是整個專案的技術風險點，先過。
3. **P2 號碼管理**：`invoices` 集合 + 每館 `invoiceState`（換捲設起始號、自動計數、作廢跳號、目前號碼顯示/校正）。
4. **P3 共用元件**：`InvoiceCheckout` + `InvoicePrinter` adapter；先接一個流程（銷售 POS）端到端。
5. **P4 四流程接線**：入場/租借/課程/定期票依序接；失敗佇列 + 補印。
6. **P5 結帳自動化**：每日結帳改讀 `invoices` 自動帶起訖/作廢；月銷售/發票明細自動產生；手動輸入降為校正退路。
7. **P6 作廢/退費連動**：取消/退費流程接作廢。

## 7. 決策紀錄 / 待辦
